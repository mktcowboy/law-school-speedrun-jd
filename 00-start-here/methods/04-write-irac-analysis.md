# How To Write Short IRAC Analysis

Reading and outlining are inputs. Written legal analysis is the output every exam, memo, and brief actually grades. `IRAC` is the basic structure.

## IRAC

- `Issue` — the precise legal question
- `Rule` — the governing rule or test, stated cleanly
- `Application` — facts mapped onto each element of the rule, both ways
- `Conclusion` — the answer

For longer writing, use `CREAC`: `Conclusion`, `Rule`, `rule Explanation`, `Application`, `Conclusion`. Same logic, with the rule explained before it is applied.

## The Part That Earns Points: Application

Most of the grade lives in `Application`. Stating the rule is table stakes. Points come from:

- applying the rule to `each element`, in order
- using the specific facts, not restating the rule
- arguing `both sides` before concluding
- not skipping the element that seems obvious

A conclusion with no application is worth almost nothing.

## Worked Shape

> Issue: Did D commit battery when he pulled the chair away?
> Rule: Battery is an intentional act causing harmful or offensive contact with another's person.
> Application: D acted intentionally by pulling the chair. Contact occurred when P hit the floor, and contact with the ground counts as contact with the person. It was harmful because P was injured. D may argue he intended only a joke, but intent to cause the contact, not the harm, is enough.
> Conclusion: D is liable for battery.

Short, ordered, element by element, both sides surfaced.

## Issue-Spotting

A fact pattern is a list of triggers. Your attack outline tells you what each trigger raises. Write one IRAC per issue, ranked by how much the facts actually fight about it. Do not write a treatise on an issue the facts barely raise.

## Time Discipline

- budget time per question before writing
- spend the first minutes spotting and ordering issues
- write tight IRACs, not essays
- a half-answered hard issue usually scores more than a perfect easy one

## Common Mistakes

- restating the rule instead of applying it
- one-sided application with no counter-argument
- burying the conclusion or omitting it
- spending all the time on the first issue
